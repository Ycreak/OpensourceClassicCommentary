Database

Regarding the database, for the fragments the following is important. An author has a name and can have written zero or more books. Every book has a title and to its ID are linked editors who have written commentaries on said book and a bibliography with an entry per citation. Most importantly, a book consists of fragments.

Every editor has a name a corresponding book and a tag whether he or she is a main editor of the text in question.

Every bibliography has multiple fields with all the information necessary to properly cite literature

A fragment has a verse number, line number, editor, content and tags whether the fragment is published and whether it is incertum, adespota or normal.

Beware! ON DELETE CASCADE is activated for these tables!

Using the unique combination Editor, book, fragment number, a referencer ID is stored in the ReferencerID table. To this number the AppCrit, Commentaries, Differences, Reconsturctions, Translation and Contexts are linked. A fragment can have one AppCrit, Commentary, Differences, Reconstruction and Translation, but multiple Contexts.

Unique Keys

Fragments: Book, Editor, fragmentName, lineName

FragmentReferencer: Book, Editor, FragmentNo

Editors: Book, editorName
